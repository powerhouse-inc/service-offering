# Commit Rules

Follow **Semantic Conventional Commits** for all commit messages.

Full specification: https://www.conventionalcommits.org/en/v1.0.0/

## Prefixes

| Prefix  | Usage                                      |
|---------|--------------------------------------------|
| `fix:`  | When fixing something (a bug)              |
| `feat:` | When adding a feature                      |
| `chore:`| When doing refactoring or maintenance work |

## Format

```
<prefix> <short description>

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
```

## Examples

```
feat: add hierarchical grid view to work breakdown editor
fix: resolve null reference in task status handler
chore: remove unused imports from editor components
```
